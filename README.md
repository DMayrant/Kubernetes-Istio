# Kubernetes-Istio
Istio uses an envoy proxy for a data plan deployed as sidecars alongside each application pod, these proxies handle all ingress and egress traffic enabling 

- Observability 🔭
- mTLS 🔐
- Traffic Control 🎛️
- Policy Enforcement and Security  

The most common service meshes are Istio, Linkerd and Console. Console is best for kubernetes and on-premise workloads. Linkerd is best for its simplicity and less operational overhead for engineering teams.
